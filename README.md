# 🐧 Linux System Administrator Practice Lab

## 🎯 Objective
This repository is designed to simulate real-world Linux system administration tasks. The goal is to practice troubleshooting, system monitoring, user management, and basic DevOps skills using Linux commands.

---

## 🧪 Scenario

You are a Junior Linux System Administrator in a company.

A production server is experiencing performance issues:
- System is slow
- Users report delays
- Some services may not be responding

Your job is to investigate, diagnose, and fix the issue using Linux commands.

---

## 🔧 Tasks

### 1. System Access
- Log into the server safely
- Check system uptime

### 2. Performance Check
- Check CPU usage
- Check memory usage
- Check disk usage

### 3. Process Analysis
- Identify top resource-consuming processes
- Kill or restart problematic processes if needed

### 4. Network Check
- Check network connectivity
- Verify open ports and services

### 5. Service Management
- Restart any failing services
- Verify service status

### 6. Reporting
Create a report explaining:
- What was the issue
- Commands used
- How you fixed it
- How to prevent it in future
